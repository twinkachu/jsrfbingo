Things you'll need to get started:

  • A copy of JSRF
  • A softmodded original Xbox or CXBX emulator
  • Quicksave mod
  • A bingo board
  • (Optional) Bingosync+ (Firefox, Chrome
  • (Optional) Regex search (Chrome, Firefox)
  • (Optional) Allgum character stat mods
  • (Optional) JSRF Multiplayer mod

Creating a bingo save

The modified game save is available in the quicksave mod - there’s a file called ‘bingo save’. You can load into the bingo save and then save your game at Roboy - this will create a bingo save that you can launch from the regular save menu in other mod XBEs. Once you do this, you can play bingo using the quicksave, IGT, and multiplayer mods. Players should be playing as gum (or as another character modified to have gum’s stats).

Generating a board

If you're using Kevingo, the site has a built-in board generator. However, if you're using Bingosync, or need access to your board JSON file, you'll need another generator tool. All of these tools will give you a JSON file that you can copy/paste into your board site.

The most user-friendly board generator is Kev360's generator site. Hit generate and copy the output. Other generator tools are available here - these offer more specific board options and alternative generation methods but will require being comfortable using python and a command line.

Setting up multi for bingo

First, follow the regular setup process for the multiplayer mod. If you're using CXBX, use Auddy's tutorial. If you're using hardware, you can follow this guide.

Once you have the mod setup, you need the multi xbe and the nocan multi files (CXBX, Xbox). The bingo xbe has a load equalizer, and the nocan files disable the multiplayer can sharing functionality.

Once everyone has the mod set up, join a multi room with your teammate on the bingo surge site. If you're using automark, send the room ID to whoever is hosting the automarker. If the multiplayer surge site is down, follow this guide to run it locally.
